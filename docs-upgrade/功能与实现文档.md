## 1. 系统概述

### 1.1 定位
Ceph 快照管理系统是一个用于管理 CloudStack 云平台中 RBD 快照的工具，支持多 Zone 环境下不同 Ceph 集群的快照管理、清理、审计等功能。

### 1.2 核心功能
| 功能 | 说明 |
|------|------|
| 卷列表管理 | 按 Zone 分组展示 CloudStack 卷，实时显示 Ceph 快照数量 |
| 快照操作 | 创建、删除、保护、取消保护 Ceph 快照 |
| 批量清理 | 批量清理旧快照，保留最新 N 份 |
| 健康检查 | 实时监控 Ceph 集群状态，判断是否允许操作 |
| 审计日志 | 记录所有操作详情，支持查询和筛选 |
| 多 Zone 支持 | 每个 Zone 可配置独立的 Ceph 集群 |

### 3.3 快照操作模块

**文件:** `ceph_snapshot_manager/routes/snapshots.py`, `ceph_snapshot_manager/services/ceph_service.py`

**功能:**
- 查看卷的所有 Ceph 快照
- 创建、删除快照
- 保护、取消保护快照

#### 3.3.1 快照列表获取
```python
# Ceph 快照列表获取
1. SSH 执行 rbd snap ls -p {pool} {volume_id} --format json
2. 解析返回的 JSON 快照信息
3. 包含字段: id, name, protected, size, timestamp
```

#### 3.3.2 创建快照
```python
def create_snapshot(self, volume_id: str, snapshot_name: str) -> OperationResult:
    pool = self.find_volume_pool(volume_id)
    snap_id = f'{pool}/{volume_id}@{snapshot_name}'
    cmd = f"rbd snap create {snap_id}"
    exit_status, stdout, stderr = self._exec_command(cmd)
    return OperationResult(success=exit_status == 0, output=stdout, error=stderr)
```

#### 3.3.3 删除快照
```python
def delete_snapshot(self, volume_id: str, snapshot_name: str, dry_run: bool = True):
    # 1. 查找卷所在存储池
    pool = self.find_volume_pool(volume_id)

    # 2. Dry Run 模式 - 只返回命令不执行
    if dry_run:
        return OperationResult(success=True, output=f'[Dry Run] {cmd}', commands=[cmd])

    # 3. 集群健康检查
    can_proceed, proceed_msg = self.check_can_proceed()
    if not can_proceed:
        return OperationResult(success=False, error=f'集群状态不允许: {proceed_msg}')

    # 4. 调用用户脚本执行删除
    script_cmd = f"{self.snap_trim_script} --disk-id {volume_id} --snap-name {snapshot_name}"
    exit_status, stdout, stderr = self._exec_command(script_cmd)

    # 5. 等待 PG 恢复
    wait_cmd = 'while ! ceph -s 2>/dev/null | grep -q "active+clean"; do sleep 5; done'
    self._exec_command(wait_cmd)

    return OperationResult(success=exit_status == 0, output=stdout, error=stderr)
```

#### 3.3.4 保护/取消保护快照
```python
# 保护快照 - 防止误删
rbd snap protect {pool}/{volume_id}@{snapshot_name}

# 取消保护 - 允许删除
rbd snap unprotect {pool}/{volume_id}@{snapshot_name}
```

### 3.4 批量清理模块

**文件:** `ceph_snapshot_manager/routes/cleanup.py`, `ceph_snapshot_manager/services/ceph_service.py`

**功能:**
- 批量选择多个卷进行快照清理
- 支持 Dry Run 模式预览
- 保留最新 N 个快照

**实现流程:**
```python
def cleanup_snapshots(self, disk_id: str, keep_count: Optional[int] = None, dry_run: bool = True):
    output_parts = []

    # [步骤1] 查找卷所在存储池
    pool = self.find_volume_pool(disk_id)

    # [步骤2] 获取快照列表
    snapshots = self.list_snapshots(pool, disk_id)

    # [步骤3] 过滤已保护快照
    valid_snapshots = [s for s in snapshots if s.get('protected') != 'true']

    # [步骤4] 按时间排序（最旧优先）
    valid_snapshots.sort(key=lambda x: x.get('timestamp', ''))

    # [步骤5] 确定保留策略
    to_delete = valid_snapshots[:-keep_count]
    to_keep = valid_snapshots[-keep_count:]

    # Dry Run 模式
    if dry_run:
        return CleanupResult(success=True, output='...', commands=[...], dry_run=True)

    # [步骤6] 集群健康检查
    can_proceed, proceed_msg = self.check_can_proceed()
    if not can_proceed:
        return CleanupResult(success=False, error=f'集群状态不允许: {proceed_msg}')

    # [步骤7] 调用用户脚本执行
    script_cmd = f"echo '{disk_id}' > /tmp/disk_ids_{disk_id}.txt && " \
                 f"{self.snap_trim_script} {dry_flag} --keep {keep_count} /tmp/disk_ids_{disk_id}.txt"
    exit_status, stdout, stderr = self._exec_command(script_cmd)

    return CleanupResult(success=exit_status == 0, output='...', commands=[script_cmd], dry_run=False)
```

### 3.5 健康检查模块

**文件:** `ceph_snapshot_manager/services/ceph_service.py`

**功能:**
- 实时监控 Ceph 集群状态
- 判断是否允许执行快照操作

#### 阻塞告警（禁止操作）
| 告警 | 说明 |
|------|------|
| PG_AVAILABILITY | PG 不可用 |
| OSD_DISK_FULL | OSD 磁盘满 |
| MON_DISK_FULL | MON 磁盘满 |
| OSD_DOWN | OSD 宕机 |
| MON_DOWN | MON 宕机 |
| CLUSTER_DOWN | 集群宕机 |

#### 阻塞 PG 状态
| 状态 | 说明 |
|------|------|
| peering | 正在协商 |
| recovering | 正在恢复 |
| backfilling | 正在回填 |
| down | PG 宕机 |
| incomplete | PG 不完整 |
| stale | PG 陈旧 |
| degraded | PG 降级 |

#### 允许告警（不阻塞）
- PG_NOT_DEEP_SCRUBBED
- PG_NOT_SCRUBBED
- DAEMON_OLD_VERSION
- NOSCRUB
- NODEEP_SCRUB

**实现:**
```python
def check_can_proceed(self) -> tuple:
    # 1. 获取集群健康状态
    health_status = self.check_health()

    # 2. 检查阻塞告警
    for alert in self.BLOCKING_ALERTS:
        if alert in checks:
            return False, f"关键告警: {msg}"

    # 3. 检查 PG 实际状态
    for pg_state in pgs_by_state:
        for blocking in BLOCKING_PG_STATES:
            if blocking in pg_state.state_name:
                return False, f"PG 状态异常: {pg_state}"

    # 4. 全部通过
    return True, "OK"
```

## 7. 核心流程图

### 7.1 快照删除流程

```
用户点击删除
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 1. 查找卷所在存储池                                         │
│    find_volume_pool(volume_id)                             │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 检查是否为最后一快照（警告提示）                           │
│    list_snapshots() → 判断 unprotected_snapshots <= 1      │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. Dry Run 模式？                                          │
│    是 → 返回预览命令，结束                                    │
│    否 → 继续                                                │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. 集群健康检查                                             │
│    check_can_proceed()                                     │
│    • 检查阻塞告警                                            │
│    • 检查 PG 状态                                            │
└─────────────────────────────────────────────────────────────┘
    │
    ├─ 不通过 → 返回错误，终止操作
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 5. 调用用户脚本执行删除                                      │
│    snap-trim.sh --disk-id {id} --snap-name {name}          │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 6. 等待 PG 恢复                                             │
│    while ! ceph -s | grep -q "active+clean"; do sleep 5; done │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 7. 记录审计日志                                             │
│    audit.add_log(action='delete_snapshot', ...)             │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
返回结果给前端
```

### 7.2 批量清理流程

```
用户点击清理
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 1. 查找卷所在存储池                                         │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 获取快照列表并过滤已保护快照                               │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 按时间排序（最旧优先）                                    │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. Dry Run 模式？                                          │
│    是 → 返回预览命令，结束                                    │
│    否 → 继续                                                │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 5. 集群健康检查                                             │
│    check_can_proceed()                                     │
└─────────────────────────────────────────────────────────────┘
    │
    ├─ 不通过 → 返回错误，终止
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 6. 调用用户脚本执行                                          │
│    snap-trim.sh [--dry-run] --keep {N} /tmp/disk_ids.txt   │
└─────────────────────────────────────────────────────────────┘
    │
    ├─ 脚本内部:                                              │
│    │ • 读取 disk_ids.txt 中的卷 ID                          │
│    │ • 获取每个卷的快照列表                                  │
│    │ • 过滤已保护快照                                        │
│    │ • 按时间排序                                            │
│    │ • 删除旧快照                                            │
│    │ • 每删除一个等待 PG 恢复                                │
│    └────────────────────────────────────────────────────────│
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 7. 记录审计日志（包含完整脚本命令）                           │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
返回详细执行结果给前端
```

### 7.3 健康检查流程

```
check_can_proceed()
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 1. 获取集群状态                                             │
│    ceph -s --format json                                   │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 检查阻塞告警列表                                          │
│    BLOCKING_ALERTS = [                                     │
│      'PG_AVAILABILITY', 'OSD_DISK_FULL', 'MON_DISK_FULL', │
│      'OSD_DOWN', 'MON_DOWN', 'CLUSTER_DOWN'                │
│    ]                                                       │
└─────────────────────────────────────────────────────────────┘
    │
    ├─ 发现阻塞告警 → return False, "关键告警: ..."
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 检查 PG 实际状态                                         │
│    BLOCKING_PG_STATES = [                                  │
│      'peering', 'recovering', 'backfilling', 'down',       │
│      'incomplete', 'stale', 'degraded'                     │
│    ]                                                       │
└─────────────────────────────────────────────────────────────┘
    │
    ├─ 发现阻塞状态 → return False, "PG 状态异常: ..."
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. 检查健康状态                                             │
│    health.status != 'HEALTH_OK'                            │
│    且存在非允许告警 → return False, "集群告警: ..."          │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
return True, "OK"
```
